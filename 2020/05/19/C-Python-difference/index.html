C语言和Python的区别

发表于 2020-05-19 分类于 小知识

C语言是可以用来编写操作系统的贴近硬件的语言，所以，C语言适合开发那些追求运行速度、充分发挥硬件性能的程序。而Python是用来编写应用程序的高级编程语言。

  1. C 是编译型语言，Python 是解释型语言

编译型语言的优点是 静态，代码不能一行一行编译执行，必须作为整个工程来编译。

优点：

  • 便于类型检查，降低运行时错误率
  • 运行时效率更高，因为编译器可以统筹各个方面，生成更优化的机器指令
  • 一经编译便可直接以机器语言再次执行，速度很快

缺点：

  • 需要大量的编译时间，不够灵活。在大项目中，需求的更改可能导致巨大的重新编译时间。

解释型语言的优点是 动态，代码的每一行可独立执行（代码块除外）。

优点：

  • 可以灵活地进行实时交互，调整正在运行的程序，进行实时、异步的调试。
  • 其适合做脚本语言。

缺点：

  • 执行时一行一行地翻译成CPU能理解的机器码，这个翻译过程非常耗时，所以很慢

  • 解释型语言难以实现静态类型检查，所以容易产生运行时错误，因此相对不适合用来做大项目。

2. Python 是原生支持面向对象范式的语言，C 不是

Python 是原生支持面向对象范式的，它有 class 的封装，有继承、多态方法。C 要实现面向对象的特性，需要一些技巧。

3. C 是弱类型、静态类型检查的，Python 是强类型、动态类型检查的

详细的解释，参考 [1]

简单概括：

C 语言

  • 弱类型

    int 可以和任何一种指针强制转换；所有的指针都可以互相转换。内存中的对象可视为拥有任何类型。

  • 静态类型检查

    C语言需要为每个变量、函数写上类型名。

Python

  • 强类型

    Python中，对象=属性+方法。每个对象都携带了类型信息。

  • 动态类型检查

    不用申明类型

总结

C Python
用来编写操作系统的贴近硬件的语言 用来编写应用程序的高级编程语言
编译型语言 解释型语言
原生支持面向过程 原生支持面向对象
必须在C中声明变量类型 无需声明变量类型
静态类型检查，弱类型 动态类型检查，强类型
有指针 没有指针
用户必须自己管理内存 有垃圾回收机制（garbage collector）
